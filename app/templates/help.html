Help

This page describes how to use {{ app_name }}, recurrence rules, admin functions, email/password reset, and common troubleshooting.

  1. Log in with your user account. Admins can create additional users under Admin → Users.
  2. Create a task under New Task. The due date is optional; if omitted, the creation time is used.
  3. Use the dashboard to track Time Left. Completing a task can optionally create the next task based on recurrence settings.
  4. Use Archived to view completed/deleted tasks and restore them back to active.
  5. Use Profile to change theme, archived auto-purge behavior, and (optionally) set an email address.

Tasks have three states: active, completed, or deleted. Completed and deleted tasks are considered archived. Archived tasks are retained for a period (configured per user) and then purged by the background scheduler.

  • Complete: marks the task completed and, if recurrence is configured, creates the next due date for the same task.
  • Delete: soft-deletes the task (it is not immediately removed from the database).
  • Archived: view completed/deleted tasks and restore them back to active.
  • Task Type: dashboard supports filtering and sorting by Task Type.
  • Admin Views: admins default to “My Tasks”. Use the Views menu for “All Tasks” or a specific user.

Recurrence determines how the due date is updated after you complete a task.

1) Post-Completion Interval

After you mark the task complete, the next due date is set to “completion time + interval”. Use duration strings like: 30m, 8h, 1d, 1d 2h, 2w.

2) Multi-Slot Daily Scheduling

The task recurs at specific times every day. Enter a comma-separated list of 24-hour times, for example: 08:00, 15:00, 23:00. When you complete the task, the next due time is the next slot that has not yet passed (or the first slot tomorrow if all have passed).

3) Fixed Clock Scheduling

Fixed clock scheduling is anchored to the calendar or to a clock interval, regardless of when you complete it.

Fixed interval examples (anchored): 6h, 1d, 2w.

Weekly rule examples:

  • Every Tuesday
  • Mon Wed Fri

Monthly rule examples:

  • 10th of every month
  • First Monday
  • Last Friday

Yearly rule examples:

  • January 5
  • Jan 5
Tip: if a recurrence input is rejected, it will display an error on the task form. Correct the interval or rule and save again.

  • Task Type: free text label used for grouping/sorting.
  • Description: optional notes shown on the dashboard and edit view.
  • Tags: comma-separated list. Tags are normalized and reused across tasks.
  • Due Date: optional. If omitted, the server uses the creation time. Dates are stored internally in UTC and displayed in the configured application timezone.

If a user has an email address set in Profile and SMTP is configured in settings.yml, Timeboard can:

  • Send hourly reminders for overdue active tasks.
  • Allow password resets from the login screen via the Forgot Email? link.
  • Allow login with either username or email address.
If email is not configured, password reset emails and reminders will not be sent.

Admin users can manage users from Admin → Users.

  • Create users: choose username, optional email, and password.
  • Toggle admin: promote/demote users between Admin and User roles.
  • Delete users: removes the user and their tasks.
  • Database export/import: available under Admin → Database as JSON.

Timeboard automatically serves a mobile-friendly layout when accessed from a mobile device. The footer includes a link to switch to the desktop layout.

Configuration is read from /data/settings.yml by default (or from the path in TIMEBOARD_SETTINGS). A sample file is included as settings.sample.yml.

app.timezone
IANA timezone name used for display and local parsing (example: America/Denver).
app.base_url
Public URL prefix when the app is behind a reverse proxy or served from a subpath.
security.session_secret
Secret used to sign session cookies. Change this to a strong random value.
security.jwt_secret
Secret used for API tokens (JWT). Change this to a strong random value.
database.path
SQLite database file path (default: /data/timeboard.db).
purge.default_days
Default retention for completed/deleted tasks for new users.
purge.interval_minutes
How often the purge job runs.
email.*
SMTP and reminder settings (enable password reset and overdue reminders).

The OpenAPI documentation is available under /docs.

  • Auth: /api/auth/token issues a JWT.
  • Tasks: /api/tasks supports filtering with tag, task_type, status, and sorting with sort. Archived tasks can be restored with POST /api/tasks/{id}/restore.
  • Users: admins can update a user (email, role) with PATCH /api/users/{user_id}.

  • Dashboard colors do not change: force-refresh the page. Static assets are versioned, but intermediate proxies can still cache aggressively.
  • Login loop: verify security.session_secret is stable across restarts and that the container clock is correct.
  • Email not sending: confirm email.enabled and SMTP settings. Check container logs for SMTP errors.
  • Timezone confusion: confirm app.timezone is correct and that your browser is not overriding time inputs unexpectedly.
  • Reverse proxy paths: when serving under a subpath, set app.base_url accordingly.
