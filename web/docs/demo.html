Demo mode

Run a public test environment safely: periodic resets, optional outbound restrictions, and clear user-facing messaging.

What demo mode does

  • Seeds the database with a demo dataset (including an admin user)
  • Optionally resets the dataset on an interval
  • Can block outbound integrations (email/webhook push) to prevent abuse
  • Shows a prominent demo warning and demo credentials on the login page
User-facing behavior
When demo mode is enabled, the login page displays: THIS IS A DEMO ENVIRONMENT AND PERIODICALLY RESET and the demo admin username/password.

Suggested demo configuration

demo:
  enabled: true
  reset_interval_minutes: 360
  disable_external_apis: true

The demo reset interval determines how frequently the database is re-seeded. Disabling external APIs prevents the demo instance from being used to send emails or arbitrary webhook traffic.

Demo credentials

Default demo admin credentials:

username: admin
password: demo

If you change the demo password, update your demo dataset or reset interval accordingly.
