API

TimeboardApp ships with a documented JSON API (OpenAPI). Use it for automation, integrations, and management workflows.

API documentation

On any running TimeboardApp instance, you can browse:

  • /docs — Swagger UI
  • /openapi.json — OpenAPI schema
Base URL
Examples below use http://localhost:8888. Replace with your deployment origin.

Authentication

The API uses a bearer token. First, exchange a username/password for an access token: 

curl -X POST "http://localhost:8888/api/auth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=YOUR_PASSWORD"

The response includes access_token. Use it as:

Authorization: Bearer <token>

Example:

TOKEN="$(curl -s -X POST "http://localhost:8888/api/auth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=YOUR_PASSWORD" | python -c "import sys, json; print(json.load(sys.stdin)['access_token'])")"

curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8888/api/auth/me"

Key endpoints

AreaPrefixNotes
Auth /api/auth Token issuance and current user
Tasks /api/tasks Create tasks, complete tasks, schedule recurrence, list by status
Tags /api/tags Manage tags and task/tag mappings
Notifications /api/notifications Notification services and user settings
Admin /api/admin Admin-only endpoints (export/import, user ops)

For exact request/response schemas, use /docs on your running instance.
