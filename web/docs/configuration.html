Configuration

TimeboardApp is configured with a YAML settings file plus optional environment variable overrides for secrets and URLs.

Settings file

The app reads a YAML file (default path: /data/settings.yml in Docker). If the file is missing, TimeboardApp creates a minimal one. A full example is included as settings.sample.yml in the repository. 

app:
  name: "TimeboardApp"
  timezone: "America/Denver"
  host: "0.0.0.0"
  port: 8888
  base_url: ""

For production deployments behind TLS, set base_url to your external URL (for example, https://timeboardapp.com for the public site or your own domain for the app instance).

Environment variables

For container deployments, it is common to keep secrets out of the YAML file and override them via environment variables. TimeboardApp supports the following overrides:

VariablePurpose
TIMEBOARDAPP_SETTINGS Path to settings.yml. Default: /data/settings.yml.
TIMEBOARDAPP_SESSION_SECRET Overrides security.session_secret (UI session cookie signing).
TIMEBOARDAPP_JWT_SECRET Overrides security.jwt_secret (API token signing).
TIMEBOARDAPP_BASE_URL Overrides app.base_url (used in external links/notifications).
TIMEBOARDAPP_PORT / PORT Overrides app.port (useful in certain orchestrators).
Compatibility
Legacy environment variable names (without the APP suffix) are still accepted for upgrades.

Database

The default database is SQLite. In Docker, store it on a volume (for example ./data mounted to /data). Backups can be done by copying the database file while the container is stopped. 

database:
  path: "/data/timeboardapp.db"

Demo mode

Demo mode is intended for public test environments. When enabled, TimeboardApp can periodically reset the demo dataset and disable outbound integrations. 

demo:
  enabled: true
  reset_interval_minutes: 360
  disable_external_apis: true

See Demo mode for operational notes and user-facing behavior.
