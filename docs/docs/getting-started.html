Getting started

Run TimeboardApp locally with Docker Compose. Defaults to http://localhost:8888.

Prerequisites

  • Docker + Docker Compose
  • A writable directory for persistent data (SQLite DB + settings.yml)
Ports
Docker Compose publishes container port 8888 to the host. If you set PORT, the host port changes.

1) Start the stack

From the repository root:

docker compose up --build

If your Compose configuration uses an external network (default: net_internal), create it once:

docker network create net_internal

2) Open the UI

Open:

  • http://localhost:8888

If you mapped a different host port (via PORT), use that instead.

3) Sign in (first run)

On first run, TimeboardApp generates an admin account and prints a one-time password to the container logs. You can read it with: 

docker compose logs -f timeboardapp

Alternatively, if you have shell access inside the container, you can reset it using the CLI:

docker compose exec timeboardapp python -m app.cli reset-admin
