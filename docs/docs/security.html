Security

Operational security guidance for self-hosting TimeboardApp.

Secrets

TimeboardApp uses two important secrets:

  • Session secret — signs UI session cookies
  • JWT secret — signs API bearer tokens

Generate strong random values for both. In production, prefer environment variables to keep secrets out of files. 

export TIMEBOARDAPP_SESSION_SECRET="..."
export TIMEBOARDAPP_JWT_SECRET="..."

Demo mode safeguards

Demo mode is convenient for public testing but should be treated as untrusted traffic. If you run a public demo, enable outbound restrictions. 

demo:
  enabled: true
  disable_external_apis: true
Reset + transparency
Demo mode surfaces a warning on the login page to reduce user confusion about periodic resets.

Reverse proxy + TLS

For Internet-facing deployments, terminate TLS at a reverse proxy and keep TimeboardApp on a private network. Ensure your proxy forwards the standard headers so the app can generate correct external links.

  • Use HTTPS (TLS) for all public access
  • Set app.base_url to the external HTTPS origin
  • Restrict admin access if needed (IP allowlists, SSO gateways)
