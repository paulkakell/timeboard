Docs · Architecture

Architecture

A small, practical architecture: FastAPI + templates, SQLite storage, background jobs for recurrence and reminders.

High-level components

Backend

FastAPI

Serves both server-rendered pages (Jinja templates) and a JSON API. Authentication is session-based for the UI and JWT-based for the API.

Storage

SQLite + SQLAlchemy

Single-file persistence on a volume. This keeps deployments simple while supporting common workloads.

Jobs

Background scheduler

Runs periodic tasks: overdue reminders, purges, and demo resets. These jobs are part of the application lifecycle.

Task recurrence model

TimeboardApp supports multiple recurrence styles:

  • Interval after completion — e.g. “every 8 hours after you complete it”
  • Fixed clock schedule — e.g. “at 08:00, 15:00, and 23:00 every day”
  • Weekly/Monthly patterns with fixed due times
  • Time slots for tasks that must occur in defined windows
Why it matters
Many task apps handle “daily” well, but struggle with sub-daily cadence. TimeboardApp treats time-of-day as a first-class scheduling concern.

Notifications

Notifications are configurable per user and can be delivered through multiple services: browser (SSE), email, generic webhooks, Gotify, ntfy, and WNS. Notification settings are stored in the database.

Webhooks include a shared secret header for validation.
